What is HackDucky?

HackDucky is Hack Club's own version of a USB rubber ducky! A USB rubber ducky is a device that looks like a regular USB drive but acts as a trojan. It pretends to be a keyboard to the computer, allowing you to automate tasks and take control of the computer you plug it into.

What You Get
  • Dual USB ports - USB-A and USB-C for versatile connectivity
  • LED indicators - Status LED plus a spare LED for your custom projects!
  • Programmable button - Add extra features to your scripts
  • Debug points - Use it as a development board for other projects!

Setup Guide

Your HackDucky ships without firmware preinstalled. Follow these steps to get started!

Step 1: Flash CircuitPython
  1. Visit the Raspberry Pi Pico 2 CircuitPython download page
  2. Download the .uf2 firmware file
  3. Plug in your HackDucky - a drive named RPI-RP2 should appear
  4. Drag and drop the .uf2 firmware into the drive
  5. Eject the HackDucky and plug it back in
  6. Success! A drive named CIRCUITPYTHON should now appear
Step 2: Install the Firmware
  1. Clone the repository: git clone https://github.com/hackclub/hackducky
  2. Open the folder and navigate to the firmware directory
  3. Copy all files from the firmware folder to the CIRCUITPYTHON drive
  4. Choose to replace existing files
  5. Delete the existing code.py file
Step 3: Add Your Script
  1. Delete any existing files in the ducks folder on your HackDucky
  2. Copy your script into the ducks folder with the extension .ducky

Time for scripting!

Ever wanted to plug in a USB device and automate a computer? HackDucky lets you do just that using a simple scripting language inspired by Ducky Script. Read on to learn the basics and start building your own scripts!

Supported Commands

  • REM : Comments for humans, ignored by the duck
  • DELAY : Pauses between actions (milliseconds)
  • STRING : Types out text like a human
  • GUI, CTRL, ALT, SHIFT: Modifier keys for shortcuts
  • ENTER : Simulates pressing Enter
  • TAB, ESCAPE, SPACE, BACKSPACE, DELETE: Special keys
  • F1-F12, Arrow Keys, HOME/END, PAGEUP/PAGEDOWN, CAPSLOCK: Navigation and function keys
  • DEFAULT_DELAY: (Optional) Sets a default delay between commands

Example: Open Notepad, type, save and close!

REM Basic example script
DEFAULT_DELAY 200

REM Open Run dialog
GUI r
DELAY 500

REM Launch Notepad
STRING notepad
ENTER
DELAY 1000

REM Type some text
STRING Hello from HackDucky!
ENTER

REM Save the file
CTRL s
STRING test.txt
ENTER
ALT F4

What's happening here?

  • GUI r opens the Run dialog
  • STRING notepad types "notepad" and hits Enter
  • STRING Hello from HackDucky! types a message
  • CTRL s saves the file as "test.txt"
  • ALT F4 closes Notepad

Pro Tips!

  • Start simple, then add more features
  • Use DELAY wisely—too fast and your computer might miss keystrokes
  • Test your script before using it on someone else's computer
  • Have fun and share your creations in the #hackducky channel on Hack Club Slack!
  • For complex projects: compile Python files to .exe, upload them publicly, then create a script to download and run them

LED Indicators

  • Triple blink: Startup sequence
  • Single blink: Working/Processing - running or starting execution
  • Rapid blinking: DuckyScript error or firmware issue

Troubleshooting

Read-only filesystem / Can't see the drive?

Your Ducky is in payload mode. Press and hold the programming pin (closest to the USB-A port) while plugging it in.

Script not working?
  • Make sure you're not in programming mode - unplug and plug back in
  • Open the debug.log file and check for ERROR lines
  • If the error mentions your script, check your syntax
  • If it's a different error, create an issue on GitHub
Keyboard not recognized?
  • Try a different USB port
  • Check the USB connection
  • Verify your target system supports USB HID devices

Submission Guidelines

Time Requirement
  • Minimum of 5 hours tracked with hackatime
  • Simple DuckyScripts can be enhanced by creating Python scripts, compiling them to executables, and downloading/running them via your DuckyScript
  • Challenge yourself to spend 5 hours on complex DuckyScript!
Originality
  • Your project must be an original idea
  • Tutorials can be referenced, but submissions should be your own creation
Submission Limit
  • You can submit multiple projects
  • You can only receive one HackDucky
HackScript Requirements
  • File format: .txt
  • Use proper Ducky Script syntax
  • Include comments with REM
  • Test your script before submitting
CAD File Requirements
  • File formats: STL or STEP preferred
  • Maximum file size: 10MB
  • Proper orientation for printing (no wasteful supports on overhangs)
  • Should fit the HackDucky board
